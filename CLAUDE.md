## Build and Test Commands

When working in this repository, use these commands:

### Quality Checks (run before committing)
```bash
npm run lint          # ESLint with auto-fix
npm run prettier      # Prettier with auto-fix
npm test              # Jest test suite
npm run build         # Full build (CJS + ESM + types)
```

### CI-Equivalent Commands
```bash
npm run lint:check      # ESLint without auto-fix (used in CI)
npm run prettier:check  # Prettier without auto-fix (used in CI)
```

### Protobuf Regeneration
```bash
buf generate         # Regenerate from buf.build/project-kessel/inventory-api
npm run prettier     # Format generated files
```

## Pre-Commit Hooks

This repository uses `rh-multi-pre-commit` (Red Hat's pre-commit framework). The hook runs automatically on `git commit`. If you need to regenerate protobuf files or run formatting, do so before committing.

## Claude Code Preferences

### When Making Changes

1. **Never edit generated files.** Files with `// Code generated by protoc-gen-ts_proto. DO NOT EDIT.` are regenerated every 6 hours by CI. Changes must go through upstream proto definitions at `buf.build/project-kessel/inventory-api`.

2. **Run quality checks before committing.** Always run `npm run lint && npm run prettier && npm test` before creating commits. CI tests on Node 20, 22, and 24 -- all checks must pass.

3. **Use relative imports in source code.** Source files use relative imports (e.g., `import { OAuth2ClientCredentials } from "../auth"`), not package imports. Package subpath imports are only for examples and consumers.

4. **Keep tests colocated.** Test files live in `__tests__/` directories next to the code they test, using the same filename (e.g., `__tests__/index.ts` tests `../index.ts`).

### Package.json Exports

When adding new public modules under `src/kessel/`, update the `exports` field in `package.json` with all three conditions:
- `require` (CommonJS)
- `import` (ESM)
- `types` (TypeScript declarations)

Plus a wildcard entry for deep imports (e.g., `./kessel/inventory/v1/*`).

### CI Pipeline Awareness

- `.github/workflows/build.yml` runs on every push/PR to main (lint, prettier, build, test on Node 20/22/24)
- `.github/workflows/buf-generate.yml` runs every 6 hours and creates PRs with regenerated protobuf files
- Both workflows must pass for PR merge
